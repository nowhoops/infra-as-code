# Summary: Apache with Ansible (Karvinen, 2026b)
### Pattern:

1. Install software  
   - `apt install apache2`

2. Configure files  
   - edit `/etc/apache2/...`

3. Restart service  
   - `systemctl restart apache2`

4. Manage configs with Ansible role structure  
   - `roles/apache2/files` for config files  
   - `roles/apache2/tasks` for automation logic  
   - `roles/apache2/handlers` for restarts  

5. Deploy site config  
   - make config files to `sites-available`

6. Enable site  
   - create symlink in `sites-enabled`

7. Apply changes  
   - restart Apache via handler or systemctl  


## What they are
- Handlers run **only when notified**
- Used to react to changes (not every run)

---
# Ansible Handlers (Ansible, 2026)

### Basic idea
1. Task runs  
2. If something changes → `notify` handler  
3. Handler runs 

---

### notify
- Tasks can trigger handlers using `notify`
- One or many handlers can be triggered

---

### Loops
- If any item changes in a loop → handler runs

---

## Key idea
- Only run actions when something changes
- Avoid unnecessary restarts
- Makes automation efficient and clean

## 4. Create web directory

nano nginx/tasks/main.yml

``` yaml
- name: Create dir
  file:
    path: "{{ web_root }}"
    state: directory
    owner: "{{ web_user }}"
    group: "{{ web_user }}"
    mode: '0755'
```
 - 755 For Execute and read rights

### Success.

## 6. Configure Nginx site

``` yml
- name: Configure nginx site
  copy:
    dest: /etc/nginx/sites-available/default
    content: |
      server {
          listen 80 default_server;
          listen [::]:80 default_server;

          root {{ web_root }};
          index index.html;

          location / {
              try_files $uri $uri/ =404;
          }
      }
```
### Success.

---

## 7. Restart Nginx

``` yaml
- name: Restart nginx
  service:
    name: nginx
    state: restarted
    enabled: yes
```
