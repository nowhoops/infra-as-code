### **Karvinen 2026: SSH Public Key – Login Without Password**
(Karvinen 2026a)

**Basics of SSH**
- Install SSH:
  ```bash
  sudo apt-get update
  sudo apt-get -y install ssh
  ```
- Enable + start service:
  ```bash
  sudo systemctl enable --now ssh
  ```
- Generate SSH keypair:
  ```bash
  ssh-keygen
  ```
  Do **not** overwrite an existing keypair.

### **Karvinen 2026: Hello Ansible**

- **Ansible = Infrastructure as Code (IaC)**  
  You define the desired state; Ansible only changes what is needed.
- **Agentless**  
  Uses SSH. Managed nodes only need SSH daemon + Python.
- **Controller vs. Managed Nodes**  
  Controller runs Ansible; connects to nodes via SSH.
- **Playbooks define roles**  
  `site.yml` assigns roles (e.g., “hello”) to groups of machines.

# Exercises

## a) Sshecrets — Install SSH Daemon and Test Login

SSH was preinstalled on Debian, so you removed it first:

```bash
sudo apt-get remove --purge openssh-server
sudo apt-get remove --purge openssh-client
sudo apt-get autoremove
```

Verify:
```bash
systemctl status ssh
which ssh
```

Reinstall:
```bash
sudo apt-get -y install ssh
sudo systemctl enable --now ssh
```

Test connection:
```bash
ssh user@ip
```

## c) Hei Ansible — Hello World Over SSH

Setup:

```bash
mkdir ~/ansible/roles/hello/tasks/ -p
cd ~/ansible
sudo apt-get update
sudo apt-get install ansible
```

Create `hosts.ini`:

```ini
[user@ip]

[all:vars]
ansible_python_interpreter=/usr/bin/python
```

Test:

```bash
ansible all -a 'uptime' -i hosts.ini
```

Create `ansible.cfg`:

```ini
[defaults]
inventory = hosts.ini
```

Create role `roles/hello/tasks/main.yml`:

```yaml
- copy:
    dest: /tmp/hello-ansible
    content: "Whoop\n"
```

Error: user lacked permission → changed to:

```yaml
dest: /home/user/hello-ansible.txt
```

Create `site.yml`:

```yaml
- hosts: all
  roles:
    - hello
```

## d) Firewall Role (UFW)

Copy project:

```bash
cp -a ansible/ ansibleUbuntuServer
mkdir ~/ansibleUbuntuServer/roles/firewall/tasks
```

Directory structure:

```
hosts.ini
ansible.cfg
firewall.yml
roles/
  firewall/
    tasks/
      main.yml
```

### First fail

Error:

```
value of default must be one of: allow, deny, reject, got: deny incoming
```

Fix:

```yaml
default: allow
direction: outgoing
```

### Second fail

Error:

```
Unsupported parameters for (ufw) module: proton
```

Fix: typo → `protocol: tcp`

### Working result

Checked with:

```bash
echo "Pass" | ssh user@ip "sudo -S ufw status verbose"
```

Output shows UFW active with rules for ports 22, 80, 443.

# Additional Notes

- Don’t remove SSH remotely or you lose connection.
- Disable public key login:
  ```bash
  mv ~/.ssh/authorized_keys ~/DISABLED_authorized_keys
  ```
- Ansible configs can be placed in a new folder and run from there.
- Always test everything you install or configure.
- Save exact error messages for debugging.
