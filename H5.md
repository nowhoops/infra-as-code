### 2.1 Getting a Git Repository
- You can start using Git by: git init
- Cloning an existing rep:o git clone <url>
- To add files (git add) and committed (git commit)

### 2.2 Recording Changes
- Files have two states:
	- Tracked (known to Git)
	- Untracked (not yet added)
#### Workflow:
1. Modify files
2. Stage changes (git add)
3. Commit changes (git commit)
##### Key commands:
```
git status
```
 → shows file states
```
git diff 
```

→ shows exact changes
##### Other actions:
- Remove files: git rm
- Rename/move files: git mv

### 2.3 Viewing History
- git log shows commit history 
##### Useful options:
```
-p → show diffs
--oneline → compact view
Date (--since, --until)
Author (--author)
Message (--grep)
Content changes (-S)
```

### 2.4 Undoing Changes
##### Amend last commit:
- git commit --amend
##### Discard local changes:
- git checkout -- <file> or git restore <file>
!!!  This deletes uncommitted work permanently!!!

### 2.5 Working with Remotes
Remotes are versions of your repo hosted elsewhere 
##### View remotes:
View remotes:
```
git remote (names)
git remote -v (with URLs)
```
##### Add/remove:
```
git remote add <name> <url>
git remote remove <name>
git remote rename <old> <new>
```
##### Fetching vs Pulling:
- git fetch <remote>
→ downloads changes without merging
- git pull
→ fetches and merges automatically
##### Pushing:
- git push <remote> <branch>
→ uploads your commits
##### Deleting:
- Local: git tag -d <tag>
- Remote: git push origin --delete <tag>
#### 2.8 
- Work with local repositories
- Track and commit changes
- View history
- Collaborate via remotes (push, pull, fetch)
- Mark releases with tags
- Customize workflow with aliases


## 1.3 Getting Started - What is Git?
### Core
- Git is a version control syustem that tracks change.Unlike many older systems, it stores data as snapshots of the entire project, not as a list of file changes.
### Key Concept
- Git stores a snapshot of all files at each commit.
- If files haven’t changed, Git just references the existing version (efficient storage).
### Mostly Local Operations
- Git can store the full project history locally.
- Most actions (view history, compare versions, commit) can be done offline.
- You can push projects when you have internet connectivity.So no need to constantly communicate with a server.
### Data is Rarely Lost
- Git mostly adds data rather 
- Once committed, it’s very hard to lose changes 
### States of Files
1. Modified – File has beenm changed but not yet saved in Git
2. Staged – File is marked to be included in the next commit
3. Committed – File is safely stored
