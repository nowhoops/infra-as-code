## B) Automatic Setup (StackOverflow 2014 / Ansible 2026)

- I used our class as a refrence for the code.


### My Code:

   ```bash
   cat adduser.yml 
   - hosts: all
     become: true
     roles:
       - adduser
   ```

```yml
/ansibleUbuntuServer$ cat ~/ansibleUbuntuServer/roles/adduser/tasks/main.yml 
- name: adduser
  vars:
    username: ansible
    ssh_key: "Your key here"
  block:
    - name: Create user
      user:
        name: "{{ username }}"
        state: present
        shell: /bin/bash
        create_home: yes

    - name: Add SSH key
      authorized_key:
        user: "{{ username }}"
        state: present
        key: "{{ ssh_key }}"

    - name: Add sudoers entry
      copy:
        dest: "/etc/sudoers.d/{{ username }}"
        content: "{{ username }} ALL=(ALL) NOPASSWD: ALL"
```

Everything worked the first time. I did this in two chunks: first SSH keys and user creation, and after that, I added the `NOPASSWD` line to `sudoers.d`. Both worked without issues.

## D) File Permission Setup (DigitalOcean, 2025)

1. **Create `file.yml`**

   ```bash
   nano file.yml
   cat file.yml 
   - hosts: all
     become: true
     roles:
       - file
   cp -r roles/app/ roles/file
   nano roles/file/tasks/main.yml
   ```

2. **Configure Multi-line File Creation:**

   ```bash
   - name: multi line
     copy:
       dest: examp.txt
       content: |
         first line
         second line
         third line
       mode: "0400"
   ```

   I got this error:

   *Mode was on the same line as the command copy. Added two spaces in front.*

   Then it went through. But the `0400` mode was incorrect. I couldn't open it with `cat`. 


   **What is `0400`?**

   - **Owner:** Read-only (r--)
   - **Group:** No permissions (---)
   - **Others:** No permissions (---)

  After changes, I updated it to `0444`:

   ```bash
   - name: multi line
     copy:
       dest: examp.txt
       content: |
         first line
         second line
         third line
       mode: "0444"
   ```

   **What is `0444`?**

   - **Owner:** Read-only (r--)
   - **Group:** Read-only (r--)
   - **Others:** Read-only (r--)
